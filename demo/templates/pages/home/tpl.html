Index / Layout

All commences in index.html, as you can see is prety empty, but that's ok because this allow you to define the layout with no restriction. In this case /templates/layout/layout.html contains the menu and the div element where all the pages are going to be included, because the Menu's content still has to be rendered the Layout javascript module has to be loaded and after that Layout.init() called, check /templates/layout/Layout.js.

PD

Is a global object that holds static references to all the the toolset, it saves you the trouble of Dom handling, file importing, configuration, pages managing, templating and some UI Helpers

Config

There is nothing written on stone, almost everything can be changed check /js/demo.init.js to have a quick idea about the definition

Templates

tpl elements that will import , process and render in that spot, HTML code provided by the attributes: src, module, and fire. Also some extra or particular behaviour can adde by overriden in the process method, and the most interesting feature, automated link between any html element of that template and its instance only by using _ or _listen

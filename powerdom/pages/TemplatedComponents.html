  • Works in a similar as partials, but are lighter and uncomplicated.
  • Can be on partials
  • Normally used as passive elements like a table.
  • Can be more than one instance of the same class per page.
  • The main idea is define the component on the html, linked to the Page class and use as a utility or helper.
  • Components are defined on the page view, works as a tag with configured attributes.
  • source defines the folder where all the files will be placed.
  • The className attribute defines the FileName pattern and the class name, FileName.js and FileName.html
  • FileName.html contain the view of the partial.
  • FileName.js is a class which its name is FileName and that extends from PartialTemplate.
  • The loaded is used to determine what event will be fire when the component is ready, normally used to link the component to Page's static var so it can work with it.
  • responsive is an optional attribute used to determine a new FileName in case the current resolution fits the media query, multiple can be described using ","

TableComponent

  • Once is loaded you must set the table headers, because the css classes on MDL are different based on the cell content type.
  • setHeader({ ColumnName: booleanIsText, ColumnName2: booleanIsText ...})
  • In my example the rest request's result is processed in the backgroud using a worker to generate the html rows and that resulting string is used in the fill method, reponsable of adding it to the table's body.

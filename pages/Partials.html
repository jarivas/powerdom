  • No page related partials are located on /partials/
  • Page related partials are normally located on /pages/partials/PageTitle/FileName/ but because is highly configurable can be placed wherever you want
  • Partials are defined on the page view, works as a tag with configured attributes
  • source defines the folder where all the files will be placed
  • The className attribute defines the FileName pattern and the class name, FileName.js and FileName.html
  • FileName.html contain the view of the partial
  • FileName.js is a class which its name is FileName and that extends from PartialTemplate
  • responsive is an optional attribute used to determine a new FileName in case the current resolution fits the media query or touchable. Media query is targeted for size and touchable in case contains elements like input type="date" or type="number".., those where natively touch devices handles it better with no javascript. Multiple rules can be chained using "," in here
  • The changed is an optional attribute used to determine what event will be fire when the partial is ready

Template descendants

  • Pages, Partials and Templated Components inherit from Template, so they once they are created using a html template (DocumentFragment) will Build (modify the inner html + Inserts the result on the node target) => Bind (the events for this new elements)
  • Partials are knottier than Pages, and involves extra steps before the setUp method
  1. listen : Set the listeners in case the partial has to wait for a signal
  2. referenceElements : Auto creates static references inside the class to elements, more info on Interactivity
  3. setUp : In essence, this method will not change from its parent class, here is where the html will be work out so is the one that normally is overwritten
  4. replace : The partials content replaces completly the partial element so, their mark up dissapear in the rendered html, this is just a note, you do not have to actively do this

Loading sequence

  1. Using the className retrieve from the server the html and js (if has not been done before)
  2. Get an instance of className, using the template node (DocumentFragment) in the downloaded html as base to be processed and inserted in the partial element defined in the page
  3. When is created, the instance, call the functions build and bind to process the html and bind the necessary events
  4. The attaches the modified html to the main element
  5. The main element is scanned for partials and templated components and process them
  6. When all the partials are inserted and ready an pageLoadedclassName is emited, and this is the right moment to do stuff like retrieve data base on filter from the server

Interactivity

* Before reading any of this, see /pages/partials/static/StaticDialog .html and .js and try to understand it reading the commented part.

Explanation

Normally the pages are not only dynamic in the content generating sense also in the interactive one, form or visual elements require references to them, so easiest way of working with this elements is having static references to them in the container class, this can be the any Template class.
When you construct a Template the reference to the object is missing because is not needed anymore, the constructor is the responsable of build and render the html in the element, and can be even multiple instances on different elements in the same page. So the rest of the methods and internal variables should be static so can be easily referenced inside the class and bypass the js function this scope problem.

Pages classes does not hold on much code because pages usually broken down to partials or components and the code is encapsulated inside of them, plus partials define the references to the dynamic elements directly in the view as element's attributes.

referenceElements

* Auto creates static references inside the class to elements

Look for all the elements inside the partial with actionableElement or referenceElement attributes and uses theirs values as variable name, this variable will contain a Powerdom instance refering element.

actionableElement will be the attribute used in the elements that you want to set events or use / set their value, content, css classes. Also you can add a listen attribute in order to set automatically the events listen="keydown:validateKey,keyup:validateDates,change:changeFilters"

reference will be the attribute used just to link the node to the classs

There is a function to get those actionableElements dynamically PartialTemplate.getActionable(id) check on /partials/form/FilterDateInterval.js line 44 to understand how is used.

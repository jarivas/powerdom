Bootsequence

  1. dependeciesLoaded : All the dependencies js files that can be loaded and initialized in this moment
  2. coreLoaded : Normally you should not have the need of subscribe to this
  3. partialsInitialized : When all the partials are loaded and Initialized is the moment when late dependencies are loaded, in this case only material should be loaded at this point because the result html is already in index.html so can be make up by the material html render.
  • The /js/core/init.js is the place where those events are listened, in case you do not want to liste then in another place, you can add your code as is indicated, my advince only listen to partialsInitialized, the others events are way too internal
  • In the console you can see the whole Bootsequence and the partials loaded each time a new page is loaded

More Info

  • First, following a required structrure defined by the html-comments-placeholders the files are loaded one after another.
  • The DOM modificator PowerDOM (PD) is first thing to load because is the base of the rest
  • The next are the static components which are only shown programatically
  • When dependeciesLoaded is fired, the PageManager will be initialized
  • PageManager will load the html representation of the page, this will be placed inside the mainElement.
  • At the same time the body partials are being load
  • When the page is ready (all the partial inside of it are ready) a partialsInitialized is fired
  • Now the just loaded material lib will render its thing
  • The next time a page is ready, it will just call a refresh material's method
